# AMUHARU 品牌與動態分類改版設計規格

## 改版目標

將現有 `KURASHI MARKET｜暮集選物所` 更名為 `AMUHARU｜安暮恆生活`，並移除「固定販售四類商品」的品牌定位與文案假設。網站仍先展示現有商品，但首頁、導覽、商品篩選、常見問題與 Meta 資訊必須依前端設定中啟用的分類自動調整，未來可由後台資料取代設定來源。

`AMUHARU` 取自 Andy 的 `A`、Mumu 的 `MU` 與 Harvey 的 `HAR`；中文「安、暮、恆」分別傳達安穩、日常與長久陪伴。品牌類型調整為「日式生活提案型綜合商店」，不再以商品分類數量定義品牌。

## 採用方案與取捨

採用「集中式商店設定 + 資料驅動畫面」方案：建立單一前端設定模組，定義品牌資訊、啟用分類識別碼與分類排列順序；所有需要分類的畫面都從此設定與商品資料取得結果。

未採用把分類保留在各 HTML 頁面的方案，因為日後增減分類仍會遺漏頁面。也不在本次建立模擬後台或遠端 JSON 服務，避免提前實作尚未確定的後台契約。設定模組的輸出介面會保持簡單，未來只需替換資料來源，不必重寫畫面元件。

## 品牌與文案策略

- 英文名稱：`AMUHARU`
- 中文名稱：`安暮恆生活`
- 品牌類型：日式生活提案型綜合商店
- 核心主張：為不同日常需要整理合適的商品與服務，不以固定分類數量作為賣點。
- 文案語氣：溫和、具體、可信任；避免「四大分類」、「四種入口」及逐一列舉所有分類的固定句型。
- 分類相關敘述：可使用目前啟用分類的名稱或數量動態產生；不需要資訊時使用「多元生活提案」、「依需求探索」等不綁數量的說法。
- SEO 與分享資訊：網站標題、Meta Description、Open Graph、結構化資料與靜態商品頁同步使用新品牌。

## 設定與資料模型

新增商店設定模組，至少包含：

```js
{
  brand: {
    englishName: "AMUHARU",
    chineseName: "安暮恆生活",
    shortDescription: "..."
  },
  enabledCategoryIds: ["used-books", "invitations", "interior", "goods"]
}
```

分類資料保留識別碼、顯示名稱、短文案、圖像或標記及排序所需資訊。`enabledCategoryIds` 是首版的暫時管理入口：

1. 陣列順序決定前台分類順序。
2. 移除識別碼即可停用分類；加入已存在於分類資料中的識別碼即可啟用。
3. 分類數量不設 2、4 或 8 的程式限制。
4. 設定含有未知識別碼時忽略該項，不造成頁面錯誤。
5. 設定為空陣列時顯示明確的「目前尚無分類」狀態，不擅自恢復全部分類。
6. 未來後台上線時，以 API 回傳資料取代此設定模組，畫面使用的分類介面維持不變。

商品列表只顯示啟用分類中的商品；直接開啟已停用分類商品的既有網址時，顯示找不到商品狀態，避免前台設定與可購買內容不一致。

## 畫面與元件調整

### 共用頁首與頁尾

共用網站標誌、無障礙標籤、頁尾介紹與版權文字改用新品牌。導覽不寫死分類名稱；分類入口由共用分類資料建立。

### 首頁

Hero 改成不依賴分類列舉的品牌主張，並替換失效的 Unsplash 圖片。新圖片必須在提交前實際確認可取得，並保留本機圖片失敗預留圖。

分類區標題採「從生活需要開始」等通用說法；分類卡片依啟用設定產生。格線使用可自動換行的響應式配置，不假設固定四張卡片。零分類時顯示空狀態；一至任意數量分類時維持合理欄寬與閱讀順序。

### 商品列表與商品詳情

分類篩選選項由啟用分類產生，「全部分類」永遠存在。搜尋、排序與分頁只操作目前可見商品。商品詳情、推薦商品及麵包屑使用相同分類來源，不顯示已停用分類的內容。

### 品牌介紹與常見問題

品牌介紹改寫為家庭日常、選擇標準與服務承諾，不列舉固定商品類別。常見問題保留通用的購物、配送、付款與退換貨內容；需要分類專屬說明時，僅呈現目前啟用分類所提供的說明資料。

### 其他靜態內容

購物車、結帳、收藏、帳戶、訂單完成、`robots.txt`、`sitemap.xml`、README、授權與專案文件中的舊品牌或固定四分類敘述一併更新。商品資料本身可保留現有四類內容，因為它們是目前展示資料，不再是網站架構限制。

## 資料流與錯誤處理

```text
商店設定 + 分類資料 + 商品資料
              ↓
        解析啟用分類
              ↓
首頁分類卡／列表篩選／可見商品／分類專屬說明
```

- 未知分類設定：略過並在開發環境以可讀訊息提示，不中斷畫面。
- 無啟用分類：分類區與商品列表顯示空狀態，搜尋與篩選仍可操作。
- 圖片載入失敗：沿用本機預留圖機制。
- 未知或已停用商品：顯示既有找不到商品畫面。
- 設定載入不需要網路；首版靜態託管仍可直接執行。

## 測試策略

以 Node.js 內建測試工具新增下列案例：

1. 使用獨立測試資料設定兩個、四個與八個分類時，解析結果保持設定順序且不受目前四類展示資料限制。
2. 未知分類識別碼會被忽略；空設定回傳空集合。
3. 商品可見性與分類篩選只包含啟用分類。
4. 品牌名稱、靜態商品頁、SEO 資訊與文件中不再出現舊品牌或「固定四類」敘述。
5. 靜態商品頁產生器可重複執行且結果一致。

瀏覽器驗證至少涵蓋首頁、商品列表、商品詳情、購物車與結帳；在 375px、768px、1200px 檢查分類卡片換行、選單操作及空狀態。另以 HTTP 請求確認新 Hero 圖片可正常取得，並檢查圖片失敗時的預留圖。

## 驗收條件

1. 所有使用者可見區域採用 `AMUHARU｜安暮恆生活`，不再出現舊品牌。
2. 品牌介紹與首頁主文案不把網站描述成固定四類商品商店。
3. 只修改 `enabledCategoryIds` 即可增減或排序前台分類，無需修改 HTML。
4. 首頁分類卡、商品列表篩選、可見商品與分類專屬說明使用同一組啟用分類。
5. 兩類、四類、八類、未知分類與零分類情境都有可執行測試。
6. 新 Hero Unsplash 圖片實際回應成功；失敗預留圖仍有效。
7. 全部自動化測試通過，主要購物流程經瀏覽器驗證。
8. 儲存庫不包含 API 金鑰、token、環境檔案或私人金鑰。

## 範圍外事項

本次不建立後台、資料庫、分類管理 API、正式會員系統或真實綠界交易。未來後台需負責分類的新增、編輯、停用與排序，並透過穩定介面提供與本次前端設定等價的資料。
