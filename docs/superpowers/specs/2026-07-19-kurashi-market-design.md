# 暮集選物所設計規格

## 產品定位

- 英文名稱：KURASHI MARKET
- 中文名稱：暮集選物所
- 類型：日式生活策展型綜合電商
- 平台：桌機瀏覽器優先，支援平板與行動裝置觸控
- 部署：純靜態網站，可部署至 Synology NAS Web Station 或其他靜態網站託管環境

暮集選物所販售二手書、喜帖、百貨雜物，並展示可預約諮詢的室內設計案例。產品以策展內容、真實材質與清楚購物流程建立信任感，不把室內設計服務偽裝成一般實體商品。

## 首版範圍

首版包含首頁、商品列表、商品詳情、購物車、結帳、訂單完成、收藏、最近瀏覽、本機示範帳戶、品牌介紹與常見問題。購物流程支援搜尋、篩選、排序、規格與數量選擇、優惠碼示範、配送、發票及三種測試付款方式。

首版不包含真實會員驗證、後端服務、資料庫、庫存同步、物流串接、電子郵件、正式優惠券服務或真實金流交易。

## 頁面架構

- `index.html`：品牌主視覺、四大分類、策展專題、熱門商品與服務特色。
- `products.html`：關鍵字搜尋、分類、價格區間、商品狀態、排序與分頁。
- `product.html?id=...`：商品圖集、規格、庫存、數量、收藏、加入購物車與推薦商品。
- `cart.html`：數量調整、移除、優惠碼示範、免運門檻與費用摘要。
- `checkout.html`：聯絡資訊、配送方式、發票選項及測試付款方式。
- `order-complete.html`：模擬訂單編號、付款狀態與後續操作。
- `favorites.html`：收藏清單與最近瀏覽。
- `account.html`：本機示範訂單、配送資訊與偏好設定；不宣稱具備真實身分驗證。
- `about.html`：品牌理念與選物原則。
- `faq.html`：購物、二手書況、客製喜帖、設計諮詢與測試付款說明。

## 商品模型

所有商品共用識別碼、名稱、分類、價格、摘要、說明、圖片、庫存、標籤與推薦關聯，並依分類增加專屬資訊：

- 二手書：書況、作者、出版社、出版年份。
- 喜帖：紙材、尺寸、最低訂購量、客製期程。
- 室內設計案例：空間類型、坪數、風格與預約諮詢動作。
- 百貨雜物：材質、尺寸、產地與庫存。

商品圖片使用不含 API 金鑰的 Unsplash CDN 網址。圖片失敗時改用本機預留圖，保持版面尺寸與替代文字。

## 技術架構

網站採混合多頁式架構：每個主要頁面使用獨立 HTML，以原生 ES Modules 共用商品資料、狀態、購物車、格式化、驗證、付款模擬與 UI 元件。CSS 使用自訂屬性管理四組主題及三組主要響應式斷點。

狀態優先儲存於 `localStorage`，包括購物車、收藏、最近瀏覽、主題與模擬訂單。若瀏覽器拒絕存取 `localStorage`，狀態層退回當次頁面工作階段的記憶體內儲存方式。

商品詳情以查詢參數載入資料，並由 JavaScript 更新標題、Meta Tag、Open Graph 與 Product 結構化資料。靜態檔案同時提供 `sitemap.xml` 與 `robots.txt`。

## 購物與付款資料流

```text
商品瀏覽 → 選擇規格與數量 → 購物車
→ 填寫配送與發票資訊 → 選擇付款方式
→ 顯示測試付款提示 → 產生本機模擬訂單
```

付款模組以 `PaymentProvider` 介面隔離 UI 與供應商行為。首版提供：

- 信用卡：只接受符合測試格式的假卡號，不保存完整卡號或安全碼。
- ATM：產生明確標示為測試用途的虛擬銀行代碼與繳費期限。
- 超商代碼：產生測試繳費代碼，不連接真實超商系統。

所有結帳與完成畫面固定顯示「未產生真實交易」。首版不向綠界送出請求、不在瀏覽器產生 CheckMacValue，也不保存或引用任何 MerchantID、HashKey、HashIV、存取金鑰或私密金鑰。未來真實整合必須由受控的 .NET API 產生付款欄位與簽章。

## 視覺方向

視覺採「日式侘寂編輯風 × 都會百貨網格」。背景以米紙白與細緻紙張感為主，搭配不完全對稱但可預測的網格。卡片使用細線與留白建立層次，不依賴滿版圓角、玻璃效果、大面積漸層或浮誇陰影。

標題採日系明朝體策略，內文採清晰的日系無襯線字體，並提供系統字體回退。商品攝影應呈現自然光、真實材質與生活痕跡，避免過度光滑或不自然的合成感。

四組可切換主題為森林綠、酒紅色、倫敦藍與聰穎黃。每組主題同步定義主色、深色文字、淺色表面、焦點框、連結與選取狀態，確保不是單純替換按鈕色彩。

## 互動與響應式設計

- 動畫以淡入、遮罩揭露與小幅位移為主，避免無目的循環動畫。
- 支援 `prefers-reduced-motion`，停用非必要位移及平滑捲動。
- 所有互動目標至少 44 × 44 CSS 像素。
- 手機版提供底部購物車捷徑與可收合篩選器。
- 以約 375px、768px、1200px 作為手機、平板與桌機主要驗證寬度。

## 無障礙要求

- 使用語意化 HTML、可見焦點與合理標題階層。
- 表單控制項具備標籤、說明、錯誤關聯及鍵盤操作。
- 錯誤摘要可定位第一個錯誤欄位，並由螢幕閱讀器讀出。
- 商品圖具有有意義的替代文字；裝飾圖使用空替代文字。
- 主題色文字與互動狀態需符合 WCAG 2.1 AA 對比要求。

## 錯誤與邊界行為

- 不存在的商品識別碼顯示友善的找不到商品狀態及返回列表動作。
- 加入購物車及數量調整不可超過庫存，並提供繁體中文提示。
- 重複送出結帳時鎖定按鈕，避免建立多筆模擬訂單。
- 圖片失敗、空搜尋結果、空購物車與儲存失敗都有專屬狀態。
- 購物車金額統一由純函式計算，避免頁面各自產生不同結果。

## 測試策略

以 Node.js 內建測試工具驗證購物車計算、庫存限制、優惠規則、表單驗證、付款模擬、資料序列化與退回記憶體內狀態。每項新增行為遵循先失敗、後通過的測試驅動開發循環。

瀏覽器驗證涵蓋首頁、商品列表、商品詳情、加入購物車、購物車修改、三種付款方式與訂單完成流程，並在 375px、768px 與 1200px 寬度檢查版面。另行檢查鍵盤操作、焦點順序、表單標籤、替代文字、對比度、降低動態效果、重新整理後狀態、錯誤商品網址及圖片失敗。

## 驗收條件

1. 網站可由本機靜態伺服器啟動，且不需要建構步驟或大型框架。
2. 四類商品皆可搜尋、篩選並開啟正確的詳情內容。
3. 實體商品可加入購物車並正確限制庫存；設計案例導向預約諮詢。
4. 購物車小計、折扣、運費與總額由共用邏輯正確計算。
5. 三種付款方式皆能完成測試流程，並全程清楚標示不產生真實交易。
6. 四組主題可切換並於重新整理後保留。
7. 自動化測試全部通過，且主要流程經瀏覽器實際驗證。
8. 儲存庫不包含憑證、token、環境檔案或私人金鑰。

## 已知限制與後續整合

純靜態首版無法提供可信任的身分驗證、伺服器端庫存、真實訂單同步或安全金流簽章。正式綠界整合需要 .NET API、安全的伺服器端設定、回呼驗證、冪等處理、資料庫交易及測試環境驗證；這些內容不在本次首版範圍內。
