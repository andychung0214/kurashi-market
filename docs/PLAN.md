# 安暮恆生活專案計畫

## 需求

建立可在純靜態環境執行的日式生活提案型綜合電商。商品分類可由暫時前端設定增減與排序；目前展示資料包含二手書、喜帖、百貨雜物與室內設計案例。網站提供商品探索、詳情、收藏、購物車、結帳、三種付款模擬與本機訂單。

## 範圍

### 包含

- 十個主要 HTML 頁面與四組可保存主題。
- 16 筆商品／案例資料及不含 API 金鑰的 Unsplash 圖片。
- 可設定任意數量的啟用分類，以及由設定衍生的首頁、篩選、商品與常見問題內容。
- 搜尋、篩選、排序、分頁、詳情、收藏、最近瀏覽、購物車與費用計算。
- 信用卡、ATM、超商代碼的安全本機模擬。
- SEO、響應式設計、無障礙基礎與可執行單元測試。

### 不包含

- 後端服務、資料庫、真實會員、庫存同步、物流、郵件與真實金流。
- 瀏覽器端綠界簽章或任何正式憑證。

## 里程碑

1. 商品資料、安全儲存與核心測試。
2. 購物車、查詢、驗證及付款模擬。
3. 視覺系統、首頁與商品列表。
4. 詳情、收藏、最近瀏覽與購物車。
5. 結帳、三種付款模擬與本機訂單。
6. SEO、文件、瀏覽器驗證與程式碼審查。

## 工作分解

- `src/js/core` 維持無 DOM 的可測試業務邏輯。
- `src/js/data` 提供目前商品資料與可由未來後台取代的暫時商店設定。
- `src/js/ui` 管理共用外框、主題與重複畫面。
- `src/js/pages` 負責頁面事件與畫面組合。
- `src/css` 依 Token、基礎、元件與頁面責任拆分。
- `tests` 使用 Node.js 內建測試工具驗證行為。

## 風險

| 風險 | 影響 | 緩解方式 |
|---|---|---|
| 純前端無法安全產生綠界簽章 | 不可執行真實付款 | 僅提供明確標示的本機模擬，文件定義 .NET API 邊界 |
| localStorage 被停用或清除 | 購物狀態遺失 | 退回記憶體內狀態，UI 明示本機限制 |
| 外部圖片無法載入 | 商品卡片失去視覺 | 使用固定尺寸與本機 SVG 預留圖 |
| 多頁重複結構不一致 | 導覽或 SEO 漂移 | 共用 JavaScript 外框及人工連結檢查 |
| 主題色對比不足 | 無障礙問題 | 使用深色主色與可見焦點框，逐主題檢查 |
| 前端分類設定與未來後台契約不一致 | 上線後需要重寫頁面 | 以單一解析介面隔離資料來源，頁面只消費解析結果 |

## 驗收條件

- 不需建構步驟即可由靜態伺服器執行。
- 兩個、四個、八個與零分類設定皆有測試，前台只呈現啟用分類與所屬商品。
- 商品數量不可超過庫存，設計案例不顯示假價格。
- 費用由共用邏輯計算，付款資料不保存完整卡號或安全碼。
- 三種付款方式皆固定顯示「未產生真實交易」。
- 四組主題與購物狀態可於重新整理後保留。
- 自動化測試、語法、HTTP、連結與敏感資訊掃描通過。
