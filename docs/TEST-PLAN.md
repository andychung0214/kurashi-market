# 暮集選物所測試計畫

## 首頁與商品列表

- [ ] 首頁四個分類連結可開啟對應商品列表。
- [ ] 首頁店主選商品皆可開啟詳情。
- [ ] 搜尋「山」至少顯示一筆商品。
- [ ] 切換為「喜帖」後只顯示喜帖。
- [ ] 價格由高至低排序正確。
- [ ] 空搜尋結果提供清除條件動作。
- [ ] 商品分頁保留既有搜尋與篩選條件。

## 後續測試區

### 商品詳情、收藏與購物車

- [ ] 商品詳情顯示分類專屬資訊與正確庫存。
- [ ] 不存在的商品識別碼顯示友善錯誤狀態。
- [ ] 室內設計案例只有預約諮詢動作，不顯示假價格與購買按鈕。
- [ ] 加入購物車後數量標記即時更新，且不可超過庫存。
- [ ] 收藏與最近瀏覽於重新整理後保留。
- [ ] 購物車可以更新數量、移除商品與套用 `KURASHI100`。
- [ ] 空購物車提供返回商品列表的動作。

### 結帳與付款模擬

- [ ] 空購物車無法進入結帳流程。
- [ ] 錯誤表單顯示摘要並將焦點移至第一個錯誤欄位。
- [ ] 信用卡只接受測試 Visa 格式，且訂單不保存完整卡號或安全碼。
- [ ] 信用卡、ATM、超商代碼皆可建立測試訂單。
- [ ] 重複點擊建立訂單時只建立一筆資料。
- [ ] 完成頁與本機帳戶固定顯示測試標示。
- [ ] 重新整理後仍可查看本機測試訂單。

## 行動裝置與響應式設計

- [ ] 375px：無水平捲動，導覽可展開，底部購物車不遮擋主要按鈕。
- [ ] 768px：完整導覽、商品網格與表單欄位排列正確。
- [ ] 1200px：內容不超過最大寬度，詳情與結帳摘要保持可讀。
- [ ] 觸控目標至少 44 × 44 CSS 像素。
- [ ] 裝置旋轉後不遺失表單或購物車狀態。

## 無障礙

- [ ] 僅使用鍵盤可完成搜尋、加入購物車與三種付款流程。
- [ ] 跳至主要內容連結可見且有效。
- [ ] 標題階層合理，每頁只有一個主要 `h1`。
- [ ] 圖片替代文字、表單標籤與錯誤關聯完整。
- [ ] 四組主題的文字、連結、焦點框符合 WCAG 2.1 AA 對比。
- [ ] `prefers-reduced-motion` 下停用非必要動態。
- [ ] `aria-live` 可讀出搜尋結果與加入購物車狀態。

## 錯誤與退化

- [ ] Unsplash 圖片失敗時顯示本機預留圖且版面不跳動。
- [ ] localStorage 被阻擋時，當次工作階段仍可使用購物車。
- [ ] 錯誤商品與錯誤訂單網址提供返回動作。
- [ ] JavaScript 主控台沒有未處理錯誤。

## 執行紀錄

核取方塊只有在實際完成對應驗證後才能勾選；未執行的項目保持未勾選。
